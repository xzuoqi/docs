## Prerequisites

For SSH login, both devices must be on the same local network, and the target device must have the SSH service enabled.

:::tip
Our system images have SSH service enabled by default. If SSH is not installed or enabled, you can follow this tutorial to set it up!
:::

## Install OpenSSH

SSH remote control requires OpenSSH to be installed. Run the following command in the board's terminal to install it:

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```
sudo apt update && sudo apt install openssh-server -y
```

</NewCodeBlock>

### Start SSH Service

Run the following command in the board's terminal to start the SSH service:

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```
sudo systemctl start ssh
```

</NewCodeBlock>

### Enable SSH Service on Boot

Run the following command in the board's terminal to enable SSH service on boot:

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```
sudo systemctl enable ssh
```

</NewCodeBlock>

### Check SSH Service Status

Run the following command in the board's terminal to check the SSH service status:

<NewCodeBlock tip={`radxa@${props?.board ?? 'device'}$`} type="device">

```
sudo systemctl status ssh
```

</NewCodeBlock>

The terminal will display information indicating whether the SSH service is enabled on boot and its current status.


## SSH Remote Control

You can use either the terminal command line or the Tabby software for SSH remote login.

<Tabs queryString="ssh-mode">

<TabItem value="Terminal Command Line">

Open a terminal on another device and enter the following command to log in remotely:

<NewCodeBlock tip="PC$" type="host">

```
ssh <user-name>@<ip-address>
# Example

ssh radxa@192.168.2.186
```

</NewCodeBlock>

Where `<user-name>` and `<ip-address>` are the actual username and IP address of the target device, respectively.

You'll need to confirm the connection by typing `yes` during the connection process.

<TabItem value="Using Tabby Software">

Tabby is a powerful cross-platform serial debugging tool that supports multiple protocols including serial port and SSH. We recommend using Tabby for SSH remote login.

#### Download Tabby

Visit the [Tabby official website](https://tabby.sh/) to download and install the Tabby software.

#### Install Tabby

Select the appropriate installation package for your operating system and hardware architecture (Windows: xxx.exe, Linux: xxx.deb, macOS: xxx.dmg) to install the Tabby software.

#### Using Tabby

Double-click the Tabby icon to launch the application.

</div>

#### SSH Configuration

① --> `Settings`: Click to open settings

② --> `Profiles & connections`: Click to configure connection options

③ --> `New profile`: Click to add a new configuration

#### Select SSH Template

Choose a base configuration as a template: Select the `SSH connection` template. You can modify parameters such as device name, IP address, and port in the subsequent configuration interface.

<div style={{ textAlign: "center" }}>
  <img
    src="/en/img/common/radxa-os/system-config/tabby-template.webp"
    style={{ width: "100%", maxWidth: "1200px" }}

After selecting the `SSH connection` template, modify the following parameters:

① --> `Name`: Set the connection name

② --> `Host`: Enter the IP address of the target device

③ --> `User`: Set the username (default is `radxa`)

④ --> `Save`: Save the configuration

The default port is `22`, which doesn't need to be changed.
